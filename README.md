## What is this?

Sisu is a polyglot repository. Not a tutorial collection. Not a framework. It is one person's proof-of-work across every layer of the stack, from `mov rax, 1` to `async fn spawn()` to `foldr (+) 0`, written to be read, studied, and stolen from.

The word *sisu* is Finnish. It means the quiet, stubborn refusal to quit when everything says you should. That is what this repo is: the same hard problems attacked over and over in different languages until the *concept* clicks, not just the syntax.

If you want to understand how memory actually works, read the C. If you want to see the same idea expressed with zero side effects, read the Haskell. If you want to watch a Rust borrow checker make you a better programmer, read the Rust. They are all here, they all talk to each other, and they all compile.

## Why This Structure

Each directory is not a hello-world collection. It is the same fundamental ideas re-expressed in the idiom natural to that language.

**Memory** — In C, you `malloc` and `free`. In Rust, ownership handles it. In Haskell, you never think about it. Reading all three back-to-back is the fastest way to actually understand what memory management *means*, not just how to do it in one language.

**Concurrency** — The `paradigms/concurrent/` directory shows the same producer-consumer pattern in threads (C), async/await (Rust), and STM (Haskell). Same problem, completely different mental models.

**Game Logic** — GoDot scripts, Rust game engines, C math games, and Lua scripts all implement interactive systems. Comparing how state machines work across these languages teaches what a state machine *is*, independent of syntax.

**Data Modeling** — Cassandra CQL for distributed NoSQL, Snowflake SQL for analytics, R for statistical models, Python for ML pipelines. Same datasets, different questions, different tools.

## Philosophy

I did not set out to write code in 15 languages. I set out to understand programming. Not *a* programming language, but the thing underneath all of them. The patterns that exist whether you write them in C or Haskell. The trade-offs that do not go away when you switch frameworks.

Every file here compiles. Every function does something real. Every comment explains *why*, not *what*. The code explains what.

This repository is the stubborn, daily work of understanding made visible.
